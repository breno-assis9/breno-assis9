## 🔄 Arquitetura Orientada a Eventos e Event Sourcing

A **arquitetura orientada a eventos** é a base dos meus sistemas. Eu aplico **Event Sourcing** para garantir que todos os estados de nossos sistemas sejam registrados como eventos imutáveis. Esses eventos são processados de forma assíncrona, o que permite a escalabilidade e a flexibilidade.

- **Desacoplamento**: A comunicação entre microsserviços ocorre através de **mensageria** (Kafka/RabbitMQ), garantindo que cada serviço possa evoluir independentemente.
- **Resiliência**: A arquitetura orientada a eventos permite que os microsserviços sejam mais resilientes a falhas.
- **Escalabilidade**: Utilizando **Docker** e **Kubernetes**, posso facilmente escalar serviços de acordo com a demanda.
